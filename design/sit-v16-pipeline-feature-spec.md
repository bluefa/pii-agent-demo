# SIT v16 — 파이프라인 콘솔 기능 명세 & 디자인 개선 리뷰

> 대상: Admin "설치/삭제 파이프라인" 뷰(노드 캔버스 + 인스펙터) 및 TargetSource 설치관리 탭 미니플로우.
> 기준: ADR-016 개정 4판(b57126a). 디자인 토큰(#0064FF · Toss · Geist) 보존, 다른 화면 불변(수술적).
> 목적: **구현 전 리뷰용** — 무엇을 구현할지와 왜 기존보다 나은지를 합의한 뒤 코드에 들어간다.

---

## 1. 현황 리뷰 (지금 무엇이 있나)

### 1.1 v15 원본 파이프라인 UI (리디자인 *전*)
- **보드**: 요약 카드 4개(진행중/외부대기/실패/완료) + **flat 테이블**(행 = 파이프라인).
- **행 펼침**: `.adm-flow` **가로 노드플로우** — 고정 178px 노드 카드 한 줄, `.adm-link` 화살표 커넥터,
  노드 상태(done/running/waiting/failed/queued) 색, `nd-edit ✎`.
- **한계**:
  - 가로 스크롤 박스 — 노드가 많으면 답답, **줌/팬/미니맵 없음**.
  - 노드를 눌러도 **상세 없음**(또는 별도 모달) → 맥락 단절.
  - 상태가 **색뿐** — "정상 장기실행 vs 문제"를 구분 못 함.
  - **장기실행 가시성·liveness·systemic health·재시도 의미·계보** 전부 없음.
  - 어휘가 **stale**(EXECUTE 등 구 모델).

### 1.2 v16 현재 (1차 리디자인 — 이미 구현·검증 완료)
- **노드 캔버스**: 팬(드래그)·줌(휠 커서기준/버튼)·맞춤·**미니맵**·dot-grid, SVG **베지어 엣지**
  (상태색 + active 흐름 애니메이션), **입출력 핸들 포트**. 이징 모션(자연스러움).
- **사이드 인스펙터**(노드 클릭): operation id(`Check.name`)·상태 pill·**errorCode 진단 칩**·`dependsOn`·
  `failCount/maxFailCount`·확인주기/TTL 편집·`latestCheck`·**머지 타임라인(attempt+check, O20)**·
  **check 집계**·**postCheck TERRAFORM_LOG**(logPointer+발췌)·**취소 / 재시도(새 run)**.
- **헤더**: slot 게이지 + 다음 확인 카운트다운.
- **ADR 정합**: 9-state→6라벨, TaskKind 3종, `slot 대기`(READY∧TERRAFORM_JOB) 배지.
- **검증**: 헤드리스 Chrome 렌더 3상태(기본/외부대기+인스펙터/실패+인스펙터) 육안 확인 완료.

### 1.3 격차 — `console-ux.md`엔 명세됐으나 v16 HTML에 아직 *미구현*

| 항목 | console-ux 절 | 현재 HTML |
|---|---|---|
| systemic health/attention 배너(보드 상단) | §4.3 | ❌ 없음 |
| 보드 행 **liveness 칩**(진행 정상/예정 대기/지연/의심) | §1.2·§4.2 | ❌ 없음 (현재 Task 텍스트만) |
| **정상 기준 대비**(expectedDuration "보통 ~25분 · 현재 22분") | §4.1 | ❌ 없음 |
| 재시도 = **"다시 실행" 확인 모달 + 실패 진단** | §5 | ⚠ 버튼만 — 현재 **무모달 즉시 리셋** |
| 취소 → **CANCELLING drain 시각화**(중단 중·slot 점유) | §6 | ⚠ 버튼만 — 현재 **즉시 CANCELLED** |
| check 집계 **"개별 폴링 펼치기"** 실제 동작 | §3.1 | ⚠ 현재 `alert()` 자리표시 |
| run **계보**(retryOf) 표시 | §5 | ❌ 없음 |

## 2. 제공할 기능 (구현 대상 — 상세)

> 각 기능: 무엇 · 어디 · 인터랙션 · (mock)데이터 · 시각 · 수용기준 · 우선순위.

### F1. systemic health / attention 배너 — *우선순위 ①*
- **무엇**: 보드 상단 조건부 배너. 실패/지연/타임아웃이 감지되면 "30분 알림 전에" 주의 환기.
- **어디**: `data-aview-panel="pipeline"` 최상단(요약 카드 위).
- **조건/카피**: 실패 N건 또는 지연(기준 초과) M건이면 →
  `⚠ 파이프라인 1건 실패 · task 1건 지연 — 확인이 필요해요   [실패 보기]`.
  (mock: Azure 실패 + GCP Subnet 지연 기준으로 노출.)
- **데이터**: 기존 시드에서 파생(실패 status, 지연 = 경과 > expected). 신규 저장 없음.
- **수용**: 실패/지연 있을 때만 노출, 없으면 숨김. 토큰 색(warning/error).

### F2. 보드 행 liveness 칩 — *우선순위 ①*
- **무엇**: 각 파이프라인 행 "현재 Task" 옆에 liveness 칩.
- **라벨**: `진행 정상`(차분) / `예정 대기`(앰버·⏱시각) / `지연`(주황·경과/기준) / `실패`(빨강).
- **데이터**: 현재 task의 state + expected 기준 파생.
- **수용**: 4종 색/문구 구분, "오래 걸림=빨강" 오학습 방지(정상은 차분색).

### F3. 정상 기준 대비 (expectedDuration) — *우선순위 ②*
- **무엇**: 진행중/외부대기 노드 메타 + 인스펙터에 `보통 ~25분 · 현재 22분 (정상)` / `현재 48분 (지연 ⚠)`.
- **데이터**: 시드 task에 `expected`(기준) 추가 + `elapsedMin` 파생. (없으면 mock 고정값.)
- **시각**: 노드 nd-meta 한 줄 + 인스펙터 "진행" 섹션. 지연 시 ⚠.
- **수용**: 30분+ 정상이 빨강 아님, 기준 초과만 `지연`.

### F4. 재시도 = "다시 실행" 확인 모달 + 실패 진단 — *우선순위 ①*
- **무엇**: `재시도(새 run)` 클릭 → 즉시 리셋(현재) ✗ → **확인 모달**.
- **모달 내용**: "이 run은 되살릴 수 없어요. 같은 구성으로 **새 파이프라인 run**을 만듭니다. 완료 단계는
  빠르게 통과해요." + **실패 진단**(실패 단계 · errorCode · 마지막 check) + `[취소] [새 run 만들기]`.
- **동작**: 확인 시 새 run 생성(mock: 새 pipeline unshift 또는 상태 리셋 + 계보 라벨).
- **수용**: 모달 고지 없이 리셋되지 않음, 실패 원인이 보임.

### F5. 취소 → CANCELLING drain 시각화 — *우선순위 ②*
- **무엇**: `취소` 클릭 → 즉시 CANCELLED(현재) ✗ → **CANCELLING(중단 중)** 단계 노출 후 수렴.
- **시각**: 실행중 노드 `중단 중 · 실행 job 종료 대기`, slot 게이지 옆 `slot 점유 중`, 보드 행 상태
  `중단 중`. (mock: 짧은 setTimeout 후 CANCELLED로 전이해 흐름을 보여줌.)
- **수용**: drain 표현 + slot 점유 사유가 보임.

### F6. check 집계 "개별 폴링 펼치기" 실제 동작 — *우선순위 ③*
- **무엇**: 현재 `alert()` → 인스펙터 내 인라인 펼침(개별 폴링 mock N건 — notable 강조).
- **수용**: 클릭 시 alert 아닌 인라인 토글.

### F7. run 계보(retryOf) — *우선순위 ③*
- **무엇**: 다시 실행으로 생성된 새 run에 `#원본 다시 실행 → #새` 라벨, 보드/인스펙터 표기.
- **수용**: 새 run이 원 run과 연결돼 보임.

## 3. 기존(v15) 대비 디자인적으로 나은 점 (왜 더 나은가)

| 차원 | 기존 v15 | v16 (목표 포함) | 사용자 가치 |
|---|---|---|---|
| **공간/탐색** | 가로 스크롤 박스(178px 한 줄) | **노드 캔버스** + 팬/줌/맞춤/미니맵 | 큰 DAG·여러 단계도 한눈에 탐색, 답답함 해소 |
| **상세 접근** | 노드=색만 / 모달 왕복 | **인라인 사이드 인스펙터**(클릭 즉시) | 보드를 벗어나지 않고 맥락 유지, 화면 왕복 0 |
| **상태 표현** | 색 5종 | 색 + 라벨 + **liveness** + **기준 대비** | "정상 장기실행 vs 문제"를 한눈에 구분(F2·F3) |
| **장애 인지** | 30분 후 알림에 의존 | **health 배너** 조기 경고(F1) | false-green(멀쩡해 보이는 장애) 방지 |
| **실패 진단** | 없음(로그 뒤짐) | **errorCode 칩 + 머지 타임라인 + terraform 로그** | "로그 고고학" 불필요, 원인 즉시 파악 |
| **재시도 의미** | 모호(이어하기 오해) | **"다시 실행 = 새 run" 정직 + 진단 모달**(F4) | 오해/무한 재실행 루프 방지 |
| **취소 피드백** | 즉시 사라짐 | **CANCELLING drain 시각화 + slot 점유**(F5) | "왜 안 줄지?"의 답을 제공 |
| **모션** | 정적/툭툭 | 이징 트랜지션·active 엣지 흐름 | 자연스럽고 부드러운 조작감 |
| **ADR 정합** | stale(EXECUTE 등) | 최신(TaskKind 3종/9상태/name/detail/errorCode) | 실제 백엔드와 일치, 혼선 제거 |

핵심 한 줄: **"상태를 색으로 칠한 가로 박스" → "조작 가능한 워크플로 캔버스 + 맥락 인스펙터 + 정상/이상 판별 신호".**

## 4. 구현 순서 & 검증

1. **순서**: F1·F2·F4(보드 가시성·재시도 — 가장 눈에 띄고 가치 큼) → F3·F5(기준대비·취소 drain) → F6·F7.
2. **검증**: 각 단계마다 **헤드리스 Chrome 렌더 → 스크린샷 육안 확인**(기본/지연/실패/취소 상태).
3. **품질 가드**: `node --check`(JS 문법), 디자인 토큰만 사용, **다른 화면 보존**, 시드 데이터 정합.
4. **커밋**: 기능 묶음별 의미 단위 커밋.

## 5. 미해결 / 가정
- F3 `expected` 기준: mock 고정값으로 시작(실서비스는 console-ux UX-O1 — 설정값→이력 p50/p95).
- F4 "새 run": 프로토타입은 상태 리셋 + 계보 라벨로 표현(실서비스는 `POST /retry` → 신규 pipeline).
- F5 drain: mock setTimeout으로 CANCELLING→CANCELLED 흐름 시연.
