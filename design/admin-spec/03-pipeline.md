# 화면 3 — 설치·삭제 파이프라인

> 공통 제약·셸·어휘는 `00-overview.md` 참조.
> 이 화면은 **2개 하위 화면**으로 구성: **3A. 보드**(목록) · **3B. 상세 모달**(워크플로 캔버스 + 인스펙터).
>
> ⚠ **유지할 디자인 자산**: "워크플로 캔버스(노드 그래프 + 인스펙터)"는 이미 검증된 핵심 디자인이다.
> 자유롭게 비주얼을 다듬되, **노드 캔버스 + 클릭 시 사이드 인스펙터**라는 구조는 유지한다.

---

## 이 화면은 무엇인가

설치·삭제 Terraform 작업을 **자동 파이프라인**으로 실행하고 현황을 모니터링한다.
파이프라인 1개 = 설치 또는 삭제 1회 실행. 내부는 여러 **Task**(단계)가 **순차 실행**되며,
일부 Task는 외부(서비스 측·인프라)의 완료를 **확인주기마다 폴링하며 대기**한다.

운영자의 핵심 질문에 답해야 한다: *지금 무엇이 돌고 있나 / 무엇이 막혔나 / 정상 장기실행인가 진짜 문제인가 / 실패 원인은 무엇인가.*

# 3A. 파이프라인 보드 (목록)

## 구성 (위→아래)

### (1) 상단 KPI + 추이
| 카드 | 예시 | 의미 |
|------|------|------|
| 진행 중 | 3 | RUNNING |
| 외부 대기 | 2 | WAIT_EXTERNAL |
| 실패 | 1 | FAILED |
| 완료(최근 7일) | 12 | DONE |

- 각 카드에 미니 추이(스파크라인) 가능(선택).

### (2) 주의 배너 (조건부 — 실패/지연 있을 때만)
- 실패 N건 또는 기준 초과 지연 M건이면 상단에 경고 배너:
  `⚠ 파이프라인 1건 실패 · task 1건 지연 — 확인이 필요해요   [실패 보기]`
- 없으면 숨김. 30분 알림 전에 조기 환기하는 목적.

### (3) 필터 / 정렬 / 시간 범위
- **상태** 칩 필터(전체/진행중/외부대기/실패/완료) — 각 칩에 건수.
- **유형**(설치/삭제), **Provider**(AWS/Azure/GCP/IDC), **검색**(TargetSource·Pipeline ID).
- **정렬**: 시작시각 desc 기본. **시간 범위**: 최근 7일 등.

### (4) 파이프라인 테이블 ← "진짜 테이블"처럼
> 과거 피드백: "카드 속 행처럼 보이지 말 것 — 어드민 대시보드다운 테이블 + 행". 컬럼 구분이 또렷한 테이블로.

| 컬럼 | 내용 | 비고 |
|------|------|------|
| **Pipeline** | 상태 노드 + `PL-82` + 서브(서비스/TargetSource) | 선두 셀, 상태색 마커 |
| 유형 | 설치 / 삭제 | |
| Provider | AWS/Azure/GCP/IDC 칩 | |
| 상태 | pill (진행중/외부대기/실패/완료/취소) | 실패 시 `EXECUTION_TIMEOUT` 같은 **errorCode 코드칩** 동반 |
| 진행 | 세그먼트 진행바 (단계별 done-ok/fail/current) | "3/4" |
| 현재 Task | 현재 단계명 + liveness | `BDC Common 실행중` / `TF권한 확인 ⏱` |
| 시작 | 06-19 08:40 | |
| 경과 | 4분 / 2일 | 정상=차분 / 지연=주황 / 실패=빨강 |

- 행 클릭 → **3B 상세 모달**.
- **liveness 신호**(정상 장기실행 vs 문제 구분): `진행 정상`(차분) / `예정 대기`(앰버·다음 확인 시각) / `지연`(주황·경과/기준) / `실패`(빨강). "오래 걸림 = 빨강" 오학습 방지 — 정상 장기실행은 차분한 색.

## 상태
- 로딩 / 빈 상태("실행 중인 파이프라인이 없어요") / 에러: 공통.

## 디자인 우선순위 (보드)
1. **막힌 것·실패가 즉시 보인다** (배너 + 상태색 + 경과색).
2. **정상 장기실행과 문제를 구별** (liveness — 30분 걸려도 정상이면 빨강 아님).
3. **테이블다운 밀도·정렬·필터** — 운영자가 원하는 조건으로 빠르게 좁힌다.

# 3B. 상세 모달 (워크플로 캔버스 + 인스펙터)

행 클릭 시 **모달**로 상세를 연다. (페이지 이동 아님 — 보드 위 오버레이)

## (1) 모달 헤더 — 제어는 맨 위
```
┌─ PL-82 · ts-az-004 설치 (Azure) ───────────── [중단]  [재시도(새 run)]  ✕ ┐
│ ⚠ PE 승인 요청 실패 · EXECUTION_TIMEOUT · 마지막 확인 06-19 14:48           │  ← 실패 시 배너
```
- **파이프라인 단위 제어 버튼을 헤더 맨 위에 배치**: `[중단]`(취소) · `[재시도(새 run)]`.
- 실패한 파이프라인이면 헤더 아래 **실패 배너**(실패 Task · errorCode · 마지막 확인 시각).
- **재시도 = 새 run**: 멈춘 파이프라인은 되살릴 수 없고 같은 구성으로 새 run을 만든다(완료 단계는 빠르게 통과). 클릭 시 이 의미를 고지하는 **확인 모달 + 실패 진단**을 거친다.

## (2) 모달 본문 = 워크플로 캔버스 (Workflow Canvas)
> 라벨은 영어로: `Workflow Canvas · N steps`, `Pipeline Workflow · PL-82`.

- Task들을 **노드 그래프**로 표현. 노드 = Task, 노드 사이 **곡선(베지어) 엣지**로 의존/순서 연결.
- 진행 중 엣지엔 흐름 애니메이션(active flow). 상태색을 엣지·노드에 반영.
- **이동(팬) 가능**: 캔버스를 드래그해 좌우/상하 이동. **줌 컨트롤·미니맵은 없음**(과거 피드백으로 제거).
- 노드 표시: 단계명 · 상태(완료/실행중/대기/외부대기/실패) · 핵심 메타(예: "보통 ~25분 · 현재 22분").
- dot-grid 배경 등 캔버스다운 표현은 자유.

## (3) 노드 클릭 → 사이드 인스펙터
노드를 클릭하면 **우측(또는 측면) 인스펙터**가 열려 그 Task의 상세를 보여준다.

인스펙터 내용:
| 섹션 | 내용 |
|------|------|
| 헤더 | operation id(작업명) · 상태 pill · **errorCode 진단 칩**(실패 시: EXECUTION_TIMEOUT 등) |
| 의존 | `dependsOn`(선행 Task) · `failCount / maxFailCount` |
| **진행** | 정상 기준 대비: `보통 ~25분 · 현재 22분 (정상)` 또는 `현재 48분 (지연 ⚠)` |
| 설정 | 확인주기 · TTL (조회 중심; 편집은 선택) |
| **CHECK 요약(집계)** | `확인 42회 · 마지막 14:48 · 실행 중` — 개별 폴링은 기본 접고, 펼치면 notable만 |
| **postCheck** | terminal 스냅샷(예: Terraform 로그 포인터 + 발췌) — 있을 때만 |

- ⚠ **Task 단위 재시작/취소 버튼은 두지 않는다**(사용자 혼란 방지). 제어는 **파이프라인 단위로 모달 헤더에서만**.
- 인스펙터는 캔버스를 가리지 않게(또는 캔버스가 인스펙터를 비켜 배치되게).

## 상태 (모달)
- 진행 중 / 외부 대기(다음 확인 카운트다운) / 실패(배너+진단) / 완료 / 취소(중단 중 → 취소됨).
- **중단(취소)**: 즉시 사라지지 않고 `중단 중 · 실행 job 종료 대기`(drain) 표현 후 취소로 수렴. slot 점유 사유 표시.

## 디자인 우선순위 (모달)
1. **제어가 위에 있다** — 중단/재시도를 헤더에서 즉시.
2. **캔버스로 흐름이 보인다** — 어디까지 됐고 무엇이 막혔는지 한눈에. (이동 가능, 줌 없음)
3. **클릭하면 맥락이 열린다** — 노드 → 인스펙터로 진단·로그까지 화면 이동 0.
4. **재시도의 정직함** — "되살리기"가 아니라 "새 run"임을 분명히.

---

## 도메인 메모 (참고)

- **Task 종류 3종**: TERRAFORM_JOB(TF 실행) · GENERAL_JOB(일반 실행) · CONDITION_CHECK(조건 확인/외부 대기).
  TERRAFORM_JOB만 동시 실행 슬롯(N-cap)을 소비 → READY인데 슬롯이 없으면 `slot 대기` 배지.
- **상태**: 내부 9상태를 6개 라벨로 묶어 표시(대기/실행중/외부대기/완료/실패/취소).
- **외부 대기**의 "아직 안 됨"은 실패가 아니라 대기(TTL로만 관리). "확인 API 호출 자체가 에러"가 fail_count 대상.
- **errorCode** 예: CALL_TIMEOUT · EXECUTION_TIMEOUT · TTL_EXPIRED · IM_REJECTED · CHECK_ERROR · DISPATCH_NO_RESPONSE.
- **재시도 정책**: 멈춘 파이프라인 재개 불가 → 같은 구성으로 **새 run** 생성(retry = new run). 완료 Task는 빠르게 통과.
- **확인주기 가드**: 외부 대기 폴링은 10분 미만 불가(특히 CONDITION_CHECK ≥10분).
- Provider별 단계(DAG)는 `02-service-management.md` 도메인 메모 참조(설치 미리보기와 동일 단계).
