# 화면 5 — 가이드 관리 (Guide CMS)

> 공통 제약·셸·어휘는 `00-overview.md` 참조.

---

## 이 화면은 무엇인가

서비스 담당자 화면(연동 프로세스)의 **단계별 가이드 콘텐츠를 관리자가 편집**하는 CMS.
각 가이드는 **한국어(ko) + 영어(en)** 두 콘텐츠를 가진다. Provider마다 단계와 가이드가 다르다.

---

## 사용자 / 맥락
- 사용자: 운영 관리자(가이드 작성 담당). 안내 문구를 바로잡거나 보강할 때 진입.
- 진입: 사이드바 "가이드 관리".

---

## 화면 구성

### (1) Provider 탭
```
[ AWS ]  [ Azure ]  [ GCP ]  [ IDC ]  [ SDU ]
```
- Provider 선택 → 해당 provider의 단계별 가이드 목록.

### (2) Step별 가이드 카드 (목록)
각 단계의 가이드를 카드로 나열.

```
┌─ Step 4 · TerraformExecutionRole 등록 가이드 ───────────── [편집 ✎] ─┐
│ 🇰🇷 ko  Role을 등록하려면 IAM 콘솔에서 …                                │
│ 🇺🇸 en  To register the role, open the IAM console …                   │
└──────────────────────────────────────────────────────────────────────┘
```

- 카드: Step 번호 · 제목 · ko/en 미리보기 · `[편집]`.
- ko/en 모두 채워졌는지 표시(둘 중 비면 `미작성 ⚠`).

### (3) 편집 (인라인 또는 모달)
- `[편집]` → ko/en 두 입력 영역(리치텍스트 또는 마크다운) + `[저장]` / `[취소]`.
- **ko/en 둘 다 필수** — 하나라도 비면 저장 비활성 + 안내.
- 저장 시 미리보기 즉시 반영.

---

## 상태
- 로딩 / 빈 상태(해당 provider에 가이드 없음) / 에러(+재시도).
- 편집 중: 변경 표시 + 저장 전 이탈 경고.
- 저장 성공: 토스트 + 카드 갱신.
- 검증 실패: ko/en 누락 시 인라인 에러.

## 디자인 우선순위
1. **무엇을 어디서 고치는지 명확** — Provider → Step → ko/en.
2. **이중 언어가 한눈에** — ko/en을 나란히, 누락을 표시.
3. **편집이 가볍다** — 카드에서 바로 편집·저장.

## 도메인 메모 (참고)
- 기존 가이드 API(`GET/PUT /admin/guides/{name}`, ko/en) 기준. 본 화면은 신규 모델 없이 배치·편집 UX만.
- Step 구성은 provider별로 다름(기존 연동 프로세스 단계 기준).
