# 화면 4 — API 현황 (API Dashboard)

> 공통 제약·셸·어휘는 `00-overview.md` 참조.

---

## 이 화면은 무엇인가

운영자가 **API가 건강한지** 빠르게 확인하는 경량 대시보드.
"전체적으로 느려졌나 / 에러가 늘었나 / 최근 무엇이 실패했나"에 답한다. (심층 APM이 아닌 운영 점검용)

---

## 사용자 / 맥락
- 사용자: 운영 관리자. 장애 의심 시 또는 주기 점검 시 진입.
- 진입: 사이드바 "API 현황".

---

## 화면 구성 (위→아래)

### (1) 기간 선택 + 새로고침
```
기간: [ 최근 24시간 ▾ ]                                        ⟳ 갱신
```
- 기간: 최근 1시간 / 24시간 / 7일 등.

### (2) 상단 KPI (4개)
| 카드 | 예시 | 의미 |
|------|------|------|
| 총 호출 | 12,408 | 기간 내 호출 수 |
| 에러율 | 0.8% | 실패 비율 (높으면 경고색) |
| 평균 응답 | 142ms | 평균 지연 |
| p95 응답 | 890ms | 상위 5% 지연 |

- 에러율·p95가 임계 초과면 경고색으로 강조(선택).

### (3) API별 통계 테이블
| 컬럼 | 예시 |
|------|------|
| API | `POST /test-connection` |
| 호출수 | 320 |
| 에러율 | 2.1% |
| 평균 | 1.2s |
| p95 | 4.8s |

- 정렬 가능(p95·에러율·호출수). 느리거나 에러 많은 API가 위로 오게.
- 에러율·p95 높은 행은 시각적으로 도드라지게(막대/색).

### (4) 최근 실패 내역
| 컬럼 | 예시 |
|------|------|
| 시각 | 06-19 09:12 |
| API | `POST /check-installation` |
| 코드 | 504 |
| 응답 | 30.0s |
| TargetSource | ts-az-004 |

- 행 클릭 → **에러 메시지 상세**(모달 또는 펼침: 요청 요약 + 에러 본문/스택 요약).
- 빈 상태: "최근 실패가 없어요 👍".

---

## 상태
- 로딩(KPI·테이블 스켈레톤) / 빈 상태 / 에러(+재시도) / 정상.

## 디자인 우선순위
1. **건강 여부가 4개 숫자로 즉시** — 에러율·p95가 정상 범위인지.
2. **문제 API가 위로** — 정렬·강조로 "느린/에러 많은" API를 먼저 보게.
3. **실패는 추적 가능** — 시각·코드·대상·메시지로 바로 조사.

## 도메인 메모 (참고)
- "API" 범위: BFF API 중심(필요 시 upstream 호출 포함 — 범위는 미확정, 시안은 BFF 기준으로 충분).
- 경량 스코프 확정: KPI 4 + API별 통계 + 최근 실패 목록.
