# 서브에이전트 주도 개발 (Subagent-Driven Development)

이 문서는 독립적인 작업들을 신선한 서브에이전트로 각각 처리하고, 각 작업 후 코드 검토를 거쳐 "고품질의 빠른 반복"을 달성하는 방법론을 설명합니다.

## 핵심 원칙

"Fresh subagent per task + review between or after tasks = high quality, fast iteration"

## 지원하는 실행 방식

### 순차 실행 (Sequential)
- 관련된 작업들을 순서대로 처리
- 각 작업마다 새로운 에이전트 파견
- 각 작업 후 코드 검토 수행
- 밀접하게 연결된 작업에 적합

### 병렬 실행 (Parallel)
- 독립적인 여러 작업을 동시에 처리
- 각 문제 영역마다 에이전트 파견
- 모든 작업 완료 후 종합 검토
- 서로 다른 파일이나 서브시스템 작업에 적합

## 순차 실행 프로세스 주요 단계

1. **계획 로드** - 작업 파일 읽기
2. **서브에이전트로 작업 실행** - 신선한 에이전트 파견
3. **작업 검토** - 코드 리뷰 에이전트 파견
4. **피드백 적용** - 문제점 해결
5. **완료 표시 및 다음 작업** - 진행
6. **최종 검토** - 전체 구현 확인
7. **개발 완료** - finishing-a-development-branch 스킬 사용

## 병렬 조사 프로세스

독립적인 여러 실패를 동시에 조사할 때 사용:

1. **독립적 영역 식별** - 문제를 그룹화
2. **집중된 에이전트 작업 생성** - 각 에이전트에 구체적 범위 할당
3. **병렬 파견** - 동시 실행
4. **검토 및 통합** - 충돌 확인 후 통합

## 주의사항 (Red Flags)

문제 상황에서는 즉시 중단하고 도움을 요청해야 합니다:
- 작업 중단 발생
- 계획의 중대한 결함
- 지시사항 이해 불가
- 검증 반복 실패

---

**마지막 조언:** 계획을 철저히 검토하고, 단계별로 정확히 따르며, 검증을 건너뛰지 말고, 막히면 추측하지 말고 질문하세요.
