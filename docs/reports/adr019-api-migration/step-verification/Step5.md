## Critical findings

1. **The real-wired Step-5 panel is ORPHANED; the live wizard mounts the demo card.** `CloudTargetSourceLayout` maps `ProcessStatus.WAITING_CONNECTION_TEST → <WaitingConnectionTestStep>` (`CloudTargetSourceLayout.tsx:8,33-34`), and `WaitingConnectionTestStep` renders `ConnectionTestCard` (the v16 demo). The legacy `ConnectionTestPanel` — which DOES wire `useTestConnectionPolling` (POST async + 4s `latest_version` poll), `getSecrets`, and `CredentialSetupModal` (credential PUT) — is mounted only by `ConnectionTestSlot.tsx`, and `ConnectionTestSlot` is referenced **nowhere live** (grep: self-only). So every "real" wiring for items 1, 2, 8, 9 is currently dead code reachable only through the orphaned slot. Items 3, 4, 10 have **no UI caller at all** (live or legacy).

2. **`ConnectionTestCard` (cloud) and `IdcStep5ConnectionTest` (IDC) are local demo simulations.** Both `runTest` = `setTimeout(…, 1800)` that flips local rows to `SUCCESS`; explicit comments call this out (`ConnectionTestCard.tsx:20,55-64,93-103`; `IdcStep5ConnectionTest.tsx:38-52,102-119`). Progress strip, Run-Test gate (`allCredsSet`), completion gate (`okCount === total`), and credential select are **all component-local state**. IDC additionally calls **`getIdcResources`** (`IdcStep5ConnectionTest.tsx:26,77` → `app/lib/api/idc.ts`) — a real call, but that's the **Step-1 resource list**, not a test-connection endpoint.

3. **`FinalLogicalDbApprovalModal` (spec §7) does not exist; completion uses `CloudReqApprovalModal` / `IdcReqApprovalModal`, which call no API.** The spec's final modal was to fetch `latest-results` (counts) and PUT `acknowledgment {confirmed:true}`. The live modals do neither — `onSubmit` only triggers a project refetch (`ConnectionTestCard.tsx:121-124`, `IdcStep5ConnectionTest.tsx:179-183`). So items 3 and 10 are jointly unimplemented at the UI.

4. **Logical-DB (items 5/6/7) has no client/route and a hard local stub.** `useLogicalDatabases` synthesizes `buildFakeDatabases`/`MOCK_TOPOLOGY` (`_components/logical-db/useLogicalDatabases.ts:11-18,51-59`); save is a toast no-op (`LogicalDbSlot.tsx:34`, `ConnectionTestCard.tsx:116-119`, `IdcStep5ConnectionTest.tsx:171-175`). The §0/§7 renamed paths (`tested-logical-databases/by-resource-id`, `excluded-databases/by-resource-id`) **are in swagger** with `resourceId` query, but nothing in the app calls them. (Plain non-`by-resource-id` variants with `agentId` query also exist in swagger: `install-v1.yaml:202-358,1824-1899`.)

5. **Schema mismatch to flag for the logical-DB PUT (item 7):** swagger `SkipLogicalDatabaseItem.skip_reason` enum = **`STG` / `DEV` / `TEMP`** (`install-v1.yaml:4364-4369`), but spec §1 documents the reason codes as **`TMP` / `STG` / `DEV`**. When item 7 is wired, use **`TEMP`** (swagger), not `TMP`. Also note field name is `skip_reason` (not `reason`) and request key is `skip_logical_database_list`.

6. **Contract layer is otherwise verbatim-correct (commit 297e872).** All 8 implemented endpoints (items 1-4, 8-10) match swagger path/method/schema exactly; the route handler is the single casing boundary (`camelCaseKeys` + hand-written normalizer, no silent `as T`); `test-connection/async` correctly sends **no body** with optional `collectorImageTag` query; acknowledgment forwards `{confirmed}`. Minor note: BFF mock `testConnection(id, body)` signature drifts from real `testConnection(id, collectorImageTag)` (`lib/bff/mock/confirm.ts:1551` vs `http.ts:285`) — path correct, arg name only. Legacy `bff.projects.testConnection: POST /projects/{id}/test-connection` (`http.ts:126`) is a **different, unused** client group, not the migrated path.
