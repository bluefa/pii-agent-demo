# ADR-003: 문서 관리 전략

## 상태
승인됨 (2026-02-02)

## 맥락
Claude Code가 전체 코드를 작성하는 환경에서 문서 관리 전략이 필요했다.

**고려사항:**
- Claude Code는 코드베이스 전체를 탐색/이해 가능
- 문서와 코드 간 동기화 부담
- 유닛 테스트가 실행 가능한 스펙 역할

## 결정
**코드 중심, 최소 문서** 전략을 채택한다.

### 유지할 문서
| 문서 | 목적 |
|------|------|
| `CLAUDE.md` | 프로젝트 규칙, 컨벤션 |
| `docs/api/*.md` | BFF API 명세 (백엔드 팀 계약) |
| `docs/adr/*.md` | 주요 설계 결정 기록 |

### 유지하지 않을 문서
| 문서 유형 | 대체 수단 |
|----------|----------|
| 구현 계획 | 완료 후 삭제 |
| 검증 결과 | 유닛 테스트 |
| 변경 이력 | Git 커밋 |
| API 응답 예시 | 테스트 코드 |

## 결과
- 문서 ↔ 코드 동기화 부담 제거
- 코드가 진실의 원천 (Single Source of Truth)
- Claude Code 새 세션에서 코드 탐색으로 검증 가능

## 관련 파일
- `CLAUDE.md` - 프로젝트 규칙
- `docs/adr/README.md` - ADR 목록
